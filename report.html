1. Project Overview

The Portal Hakam JKSNJ is a web-based case management system designed to digitise and streamline the arbitration process (Hakam) under the Kaedah-Kaedah Hakam (Negeri Johor) 2022. The system manages case registration, session reports, and all 9 statutory forms (Borang 1–9) under Jadual Pertama.

This report documents the development work completed during the current sprint, covering the migration from a browser-local prototype to a fully functional local backend system.

2. Work Completed

2.1 Local Backend Server

A production-capable local server was built using Node.js + Express.js with a SQLite database. The entire backend lives in a single file (server.js) for simplicity and easy maintenance.

2.5 Frontend Migration (localStorage → API)

The original prototype stored data in localStorage (browser-only). All data operations were migrated to call the backend API:

  • Login now calls POST /api/auth/login with bcrypt password verification
  • Session is automatically restored on page refresh via GET /api/auth/me
  • All CRUD operations (create, read, update, delete) use fetch() to the API
  • Multiple users on different computers now see the same live data

2.6 File Upload (Laporan Tahkim & Borang)

Both the Laporan Tahkim form and all Borang 1–9 forms now support document attachments:

  • Accepted formats: PDF, DOC, DOCX, JPG, JPEG, PNG
  • Maximum file size: 10 MB per upload
  • Files stored in uploads/ folder on the server
  • Attachment link shown in Laporan list view for easy retrieval

6. Known Limitations (Demo Scope)

  • Passwords for demo accounts are visible in server.js — should be changed before any real use
  • No HTTPS — suitable for local/LAN only; requires SSL certificate for internet deployment
  • No automated backup — data/jksnj.db and uploads/ should be backed up manually
  • No user management UI — account changes require editing server.js and restarting
  • Borang 1–9 fields are approximations — pending official form specifications

7. Future Recommendations (If Moving to Production)

  • Cloud deployment — Host on a server or cloud VM accessible to all JKSNJ branches
  • HTTPS — Required for any internet-facing deployment (Let's Encrypt for free SSL)
  • Automated backups — Schedule daily backup of jksnj.db to an external drive or cloud storage
  • Audit log — Record who created/modified/deleted each record (important for court records)
  • MyKad / SSO integration — Authentication via government identity system
  • Email notifications — Alert Hakam when approaching 30-day deadline
